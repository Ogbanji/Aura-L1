# Avalanche Subnet Integration

AURA-L1 is designed to run as a custom Virtual Machine (VM)
within the Avalanche Subnet architecture.

## Why Avalanche
- Subnet sovereignty
- High throughput
- Fast finality
- Mature validator ecosystem

## VM Architecture
AURA-L1 provides a VM scaffold compatible with AvalancheGo:
- Custom state transition logic
- Custom consensus rules
- Native subnet deployment

## Subnet Deployment Flow
1. Compile AURA-L1 VM
2. Register VM with Avalanche network
3. Create a dedicated subnet
4. Launch validators
5. Activate chain governance

## Validator Requirements
- Avalanche validator node
- Subnet participation
- AURA-L1 staking requirements

## Ecosystem Compatibility
AURA-L1 can interoperate with:
- Other Avalanche subnets
- Cross-chain bridges
- Tooling in the Avalanche ecosystem
